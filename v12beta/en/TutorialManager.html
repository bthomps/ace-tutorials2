  • Learn some basics about Integration Servers and how to use them

    5 minutes.

    Tutorial Topics

    • Creating a Standalone Integration Server
    • Creating Integration Servers which are owned by an Integration Node
    • Connecting to an Integration Server
    • Deploying to an Integration Server

    Learning outcomes

    • Understand the use of, and concepts behind, ACE Integration Servers in v11
    • Understand how to create, connect to and deploy a simple flow to an Integration Server

    Overview

    An Integration Server is used to provide an isolated runtime environment for a set of deployed message flows and resources. Each integration server runs as a unique operating system process in a separate address space. Previous versions of this software product (IBM Integration Bus v10, IBM Integration Bus v9, WebSphere Message Broker v8 etc.) have also included integration servers, but App Connect Enterprise v11 is the first version to allow Integration Server processes to be entirely stand-alone, and defined independently from an integration node. An integration node provides a higher level operating system process, as part of a wider process hierarchy, which is designed to look after the integration servers which are associated with the integration node. In the past integration nodes have also taken on other responsibilities such as executing new deployments, configuring updates to the server processes and serving the product's administrative REST API.

    If you are planning to run the ACE software directly on a physical machine or within a virtual machine image, then you are advised to define Integration Servers under an Integration Node.

    If you are planning to run the ACE software in conjunction with a container framework such as Kubernetes or IBM Cloud Private, then it is the responsibility of this framework to ensure that the servers remain running (or are restarted appropriately) so in this situation using independent integration servers would be the better choice.

    Independent Integration Servers can also be very useful to quickly get up and running with ACE software, especially if you are working in a development phase of your project or trying out the product for the first time. This tutorial will demonstrate how to create an Integration Server under an Integration Node and as an independent process.
